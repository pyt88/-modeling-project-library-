# Требования к системе управления библиотекой

## 1. Цели системы
Автоматизация процессов выдачи, возврата и учёта книг в библиотеке.

## 2. Пользователи
| Роль | Описание |
|------|----------|
| Читатель | Может искать книги, брать и возвращать |
| Библиотекарь | Выдаёт и принимает книги, добавляет новые |
| Администратор | Управляет пользователями и ролями |

## 3. Функциональные требования

### 3.1. Читатель
- Поиск книги по названию, автору или жанру
- Просмотр статуса доступности книги
- Взятие книги на абонемент
- Возврат книги
- Просмотр своего книжного долга

### 3.2. Библиотекарь
- Выдача книги читателю
- Приём возвращённой книги
- Добавление новой книги в каталог
- Удаление списанной книги
- Просмотр списка должников

### 3.3. Администратор
- Добавление/удаление пользователей
- Назначение ролей (читатель/библиотекарь)
- Просмотр логов операций

## 4. Нефункциональные требования
- Доступ через веб-интерфейс
- Время отклика поиска < 2 секунд
- Поддержка до 10 000 книг и 1000 пользователей
